Promise - an object representing the eventual completion or failure of an asynchronous operation. It allows you to handle asynchronous code in a more readable and maintainable way.

A Promise can be in one of three states:
  • Pending: The initial state, neither fulfilled nor rejected.
  • Fulfilled: The operation completed successfully.
  • Rejected: The operation failed.

Async/Await - a syntax for handling asynchronous operations in a more synchronous, readable way. The async keyword makes a function return a Promise, and await pauses the function execution until the Promise is resolved.

  • async: Defines a function as asynchronous and makes it return a Promise.
  • await: Pauses the execution of the async function until the Promise is resolved.

HTTP Methods - define the action to be performed on a resource in an HTTP request.

  • GET: Fetches data from the server.
  • POST: Sends data to the server to create a resource.
  • PUT: Updates an existing resource on the server.
  • DELETE: Deletes a resource from the server.

Fetch API - provides a way to make HTTP requests from JavaScript. It returns a Promise and allows you to interact with RESTful services by making network requests like GET and POST.

  • fetch() is used to make a network request.
  • It returns a Promise that resolves to the Response object representing the response to the request.
  • You can handle the response asynchronously using .then() or async/await.

Node.js - a JavaScript runtime that allows you to run JavaScript outside of the browser, typically for server-side applications.

Express - a minimal and flexible Node.js web application framework that provides a robust set of features for web and mobile applications.

  • dependency - any package your project needs to run. For instance, if you’re building a web server with Express, Express is a dependency for your project.

npm (Node Package Manager) - the default package manager for Node.js. It is used to install, share, and manage dependencies (libraries or packages) that your Node.js projects rely on. npm helps you easily manage the code you use in your projects and simplifies installing libraries, ensuring compatibility, and automating package version management.

  • package - any code or module that can be installed via npm. For example, Express, Lodash, and Axios are all packages available on npm.
  • dependency - any package your project needs to run. For instance, if you’re building a web server with Express, Express is a dependency for your project.

npm Commands

  • npm init - Initializes a new Node.js project and creates a package.json file, which contains metadata and dependencies for your project.
  • npm install or npm i - Installs the dependencies listed in your package.json file.
  • npm install - Installs a specific package and adds it to your node_modules directory.

package.json

  • contains metadata about your project, such as its name, version, description, and dependencies.
  • contains scripts, which allow you to define custom commands. For example, you might define a start script to run your server.
  • scripts - This section contains custom commands you can run using npm run . For instance, npm run start would run node server.js, starting the Express server.

.gitignore

  • A file used to tell Git which files or directories to ignore and not track in version control. This is especially useful for files that are automatically generated by your development tools, such as build artifacts, logs, dependency directories (node_modules), and configuration files that contain sensitive information like passwords or API keys.
