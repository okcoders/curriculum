Module 3 Week 4 — External APIs and Deployment

This week we integrate a currency API and deploy our full-stack app to Vercel.

Calling External APIs from Node

Node.js 18+ has built-in fetch. Use it to call external APIs:

const response = await fetch("https://api.frankfurter.app/latest?from=USD");
const data = await response.json();
console.log(data.rates);

Frankfurter Currency API

Free currency exchange API — no API key needed!

// Get exchange rates from USD to EUR and GBP
const url = "https://api.frankfurter.app/latest?from=USD&to=EUR,GBP";
const response = await fetch(url);
const data = await response.json();

console.log(data.rates.EUR);  // e.g., 0.92
console.log(data.rates.GBP);  // e.g., 0.79

Add Currency Endpoint

app.get("/api/currency/:from/:to", async (req, res) => {
  const { from, to } = req.params;
  const url = `https://api.frankfurter.app/latest?from=${from}&to=${to}`;
  
  try {
    const response = await fetch(url);
    const data = await response.json();
    res.json(data);
  } catch (error) {
    res.status(500).json({ error: "Failed to fetch rates" });
  }
});

Deploy to Vercel

Step 1: Prepare Backend

Create vercel.json in your project root:

{
  "version": 2,
  "builds": [
    { "src": "index.js", "use": "@vercel/node" }
  ],
  "routes": [
    { "src": "/(.*)", "dest": "index.js" }
  ]
}

Step 2: Push to GitHub

git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/your-username/expense-api.git
git push -u origin main

Step 3: Deploy on Vercel

  1. Go to vercel.com and sign in with GitHub
  2. Click "Add New Project"
  3. Import your GitHub repository
  4. Add environment variables (see below)
  5. Click "Deploy"

Environment Variables in Production

In Vercel dashboard: Project Settings → Environment Variables

Name Value
SUPABASE_URL https://xxxxx.supabase.co
SUPABASE_KEY your-anon-key
Important: Your .env file only works locally. You MUST add env vars in Vercel's dashboard for production!

Update CORS for Production

Add your production frontend URL to allowed origins:

app.use(cors({
  origin: [
    "http://localhost:5500",
    "https://your-frontend.vercel.app"
  ]
}));

Update Frontend API URL

In your frontend, update the API URL for production:

// Simple approach
const API_URL = window.location.hostname === "localhost"
  ? "http://localhost:3000/api"
  : "https://your-backend.vercel.app/api";

// Or use a config object
const config = {
  apiUrl: window.location.hostname === "localhost"
    ? "http://localhost:3000/api"
    : "https://your-backend.vercel.app/api"
};

Deploy Frontend

  1. Push frontend code to a separate GitHub repo
  2. Import in Vercel (same process as backend)
  3. No env vars needed for static frontend
  4. Get your frontend URL
  5. Add it to backend CORS origins
  6. Redeploy backend

Common Mistakes

Mistake Problem Fix
fetch not defined Old Node version Use Node 18+ (Vercel default)
localhost in production Hardcoded URL Use environment detection
Env vars undefined Not added to Vercel Add in Vercel dashboard
CORS error in production Missing origin Add production URL to cors origins
