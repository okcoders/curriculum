Module 3 Week 2 — Fetch API and CORS

This week we connect our frontend to our backend using the Fetch API and handle cross-origin requests with CORS.

The Fetch API

Fetch is the modern way to make HTTP requests from JavaScript in the browser.

GET Request

async function getExpenses() {
  const response = await fetch("http://localhost:3000/api/expenses");
  const data = await response.json();
  return data;
}

POST Request

async function createExpense(expense) {
  const response = await fetch("http://localhost:3000/api/expenses", {
    method: "POST",
    headers: {
      "Content-Type": "application/json"
    },
    body: JSON.stringify(expense)
  });
  return await response.json();
}
Two things required for POST:
  1. Content-Type: application/json header
  2. JSON.stringify() the body object

PUT Request

async function updateExpense(id, updates) {
  const response = await fetch("http://localhost:3000/api/expenses/" + id, {
    method: "PUT",
    headers: {
      "Content-Type": "application/json"
    },
    body: JSON.stringify(updates)
  });
  return await response.json();
}

DELETE Request

async function deleteExpense(id) {
  await fetch("http://localhost:3000/api/expenses/" + id, {
    method: "DELETE"
  });
}

CORS (Cross-Origin Resource Sharing)

When your frontend and backend are on different origins (different ports count!), the browser blocks requests by default for security.

Example:

  • Frontend: http://localhost:5500
  • Backend: http://localhost:3000

These are different origins → CORS error!

Fix: Add CORS Middleware

# Install the cors package
npm install cors
const cors = require("cors");

// Allow all origins (good for development)
app.use(cors());

// Or specify allowed origins (better for production)
app.use(cors({
  origin: ["http://localhost:5500", "https://myapp.vercel.app"]
}));

Error Handling

Always wrap fetch calls in try/catch and check response.ok:

async function getExpenses() {
  try {
    const response = await fetch("http://localhost:3000/api/expenses");
    
    if (!response.ok) {
      throw new Error("Failed to fetch: " + response.status);
    }
    
    const data = await response.json();
    return data;
  } catch (error) {
    console.error("Error:", error);
    return [];
  }
}

Response Properties

const response = await fetch(url);

response.ok       // true if status 200-299
response.status   // status code (200, 404, etc.)
response.json()   // parse body as JSON (returns Promise!)

Replacing localStorage with API

Before (localStorage)

// Load
const expenses = JSON.parse(localStorage.getItem("expenses")) || [];

// Save
localStorage.setItem("expenses", JSON.stringify(expenses));

After (API)

// Load
const expenses = await getExpenses();

// Save (create new)
await createExpense(newExpense);
const expenses = await getExpenses();
renderExpenses(expenses);

Loading States

async function loadExpenses() {
  const container = document.querySelector("#expense-list");
  container.innerHTML = "Loading...";
  
  try {
    const expenses = await getExpenses();
    renderExpenses(expenses);
  } catch (error) {
    container.innerHTML = "Error loading expenses";
  }
}

Common Mistakes

Mistake Problem Fix
CORS error Backend missing CORS Add cors middleware to Express
Body not received Missing header or stringify Add Content-Type + JSON.stringify()
Promise <pending> Missing await Add await before response.json()
Empty body on server Missing middleware Add app.use(express.json())
