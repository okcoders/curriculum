Module 2 Week 1 — Variables and Strings

This week covers the building blocks of JavaScript: how to store data in variables and work with text.

Variable Declaration

JavaScript has three ways to declare variables:

// let — can be reassigned
let count = 0;
count = 1;  // OK

// const — cannot be reassigned
const name = "Alice";
name = "Bob";  // Error!

// var — old way, avoid using
var old = "don't use this";
Rule of thumb: Use const by default. Only use let when you need to reassign the variable later.

Primitive Data Types

Type Example Notes
String "hello" or 'hello' Text wrapped in quotes
Number 42 or 3.14 Integers and decimals
Boolean true or false Yes/no values

let message = "Hello";     // string

Type Coercion

JavaScript automatically converts types in certain situations:

// String + Number = String (concatenation)
"Hello " + 42        // "Hello 42"
"3" + 4              // "34" (not 7!)

// Comparisons
"5" == 5             // true (loose equality)
"5" === 5            // false (strict equality)
Always use === (strict equality) to avoid unexpected type conversions.

String Methods

Strings have built-in methods for manipulation:

const text = "  Hello World  ";

// Remove whitespace
text.trim()                    // "Hello World"

// Change case
text.toLowerCase()             // "  hello world  "
text.toUpperCase()             // "  HELLO WORLD  "

// Get length
text.length                    // 15

// Access characters
text[0]                        // " "
text.charAt(2)                 // "H"

// Find position
text.indexOf("World")          // 8
text.indexOf("xyz")            // -1 (not found)

// Extract parts
text.slice(2, 7)               // "Hello"

// Split into array
"a,b,c".split(",")             // ["a", "b", "c"]

// Check contents
"hello".includes("ell")        // true
"hello".startsWith("he")       // true
"hello".endsWith("lo")         // true
Important: String methods return NEW strings. They don't change the original. 
const name = "alice";
name.toUpperCase();     // Returns "ALICE" but name is still "alice"
const upper = name.toUpperCase();  // Save it!

Template Literals

Use backticks (`) for strings with variables:

const name = "Alice";
const age = 25;

// Old way (concatenation)
const msg1 = "Hello, " + name + "! You are " + age + ".";

// New way (template literal)
const msg2 = `Hello, ${name}! You are ${age}.`;

// Expressions inside ${}
const msg3 = `Next year you'll be ${age + 1}`;
Common mistake: Using regular quotes instead of backticks 
"Hello, ${name}"   // Wrong! Outputs: Hello, ${name}
`Hello, ${name}`   // Correct! Outputs: Hello, Alice

Working with Numbers

const price = 4.5;

// Format to 2 decimal places
price.toFixed(2)              // "4.50" (returns string!)

// Convert string to number
parseInt("42")                // 42
parseFloat("3.14")            // 3.14
Number("42")                  // 42

// Check if valid number
isNaN("hello")                // true
isNaN(42)                     // false

Expense Tracker Preview

This week we're storing expense data in variables:

const description = "Coffee";
const amount = 4.50;
const category = "Food";

// Format for display
const currencyString = "$" + amount.toFixed(2);  // "$4.50"

// Build display text
const displayText = `${description} - ${currencyString} (${category})`;
// "Coffee - $4.50 (Food)"

// Get today's date
