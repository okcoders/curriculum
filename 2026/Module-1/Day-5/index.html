Flexbox

Flexbox is a one-dimensional layout system — arrange items in a row OR column.

.container {
    display: flex;
}

This makes the container a flex container and its direct children become flex items.

Flex Direction

flex-direction: row;         /* default: left to right */
flex-direction: row-reverse;  /* right to left */
flex-direction: column;       /* top to bottom */
flex-direction: column-reverse; /* bottom to top */

Gap

gap: 20px;           /* same gap row and column */
gap: 10px 20px;      /* row-gap column-gap */
row-gap: 10px;
column-gap: 20px;

Note: gap replaces the need for margins on flex items.

Flex Item Properties

/* On the children, not the container */
flex-grow: 1;    /* grow to fill space (0 = don't grow) */
flex-shrink: 1;  /* shrink if needed (0 = don't shrink) */
flex-basis: 200px; /* starting size before grow/shrink */

/* Shorthand */
flex: 1;         /* flex-grow: 1, flex-shrink: 1, flex-basis: 0 */
flex: 0 0 200px; /* don't grow, don't shrink, 200px wide */

/* Align single item differently */
align-self: center;

Question: You want to center a div both horizontally AND vertically in its parent. What CSS do you need?

Answer:

.parent {
    display: flex;
    justify-content: center;  /* horizontal */
    align-items: center;      /* vertical */
    height: 100vh;            /* parent needs height! */
}


Positioning

The position property controls how an element is placed in the document.

Static (default)

position: static;

Normal document flow. top, right, bottom, left have no effect.

Relative

position: relative;
top: 20px;
left: 30px;

Offset from its normal position. Still takes up original space in the flow.

I'm offset 20px down, 30px right

Absolute

position: absolute;
top: 0;
right: 0;

Removed from flow. Positioned relative to nearest positioned ancestor (not static). If none, uses the viewport.

I'm in the top-right corner

Common pattern: Parent with position: relative, child with position: absolute.

.card {
    position: relative;  /* establishes positioning context */
}

.card .badge {
    position: absolute;
    top: -10px;
    right: -10px;
}

Fixed

position: fixed;
top: 0;
left: 0;
width: 100%;

Removed from flow. Positioned relative to the viewport. Stays in place when scrolling.

Use for: Sticky headers, floating buttons, modals.

Sticky

position: sticky;
top: 0;

Hybrid of relative and fixed. Acts relative until you scroll past it, then sticks.

Use for: Headers that stick when scrolled, table headers.

Z-Index

z-index: 100;

Controls stacking order. Higher values appear on top. Only works on positioned elements (not static).

Question: You have a dropdown menu that appears behind other content. How do you fix it?

Answer: Add position: relative (or absolute) and z-index: 10 (or higher than surrounding elements).



Typography

Text styling properties for readability and design.

Font Properties

font-family: 'Helvetica Neue', Arial, sans-serif;
font-size: 16px;
font-weight: 400;       /* 100-900, or normal/bold */
font-style: italic;
line-height: 1.5;       /* multiplier of font-size */
letter-spacing: 0.5px;
word-spacing: 2px;

Text Properties

text-align: left | center | right | justify;
text-decoration: none | underline | line-through;
text-transform: uppercase | lowercase | capitalize;
text-indent: 2em;       /* first line indent */
white-space: nowrap;    /* prevent wrapping */
overflow: hidden;
text-overflow: ellipsis; /* show ... when text overflows */

Truncate Text with Ellipsis

.truncate {
    white-space: nowrap;
    overflow: hidden;
    text-overflow: ellipsis;
    width: 200px;  /* needs a width */
}
This is a very long text that will be truncated with an ellipsis

Web Fonts

/* Google Fonts - add to <head> */
<link href="https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap" rel="stylesheet">

/* Then use in CSS */
body {
    font-family: 'Roboto', sans-serif;
}

Font Stack Best Practices

/* Always end with a generic fallback */
font-family: 'Custom Font', 'Fallback Font', sans-serif;

/* Common stacks */
font-family: Georgia, 'Times New Roman', serif;
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif;


Attribute Selectors

Target elements based on their attributes.

/* Has attribute */
[disabled] { opacity: 0.5; }

/* Exact match */
[type="text"] { border: 1px solid #ccc; }

/* Starts with */
[href^="https"] { color: green; }

/* Ends with */
[href$=".pdf"] { background: url('pdf-icon.png'); }

/* Contains */
[class*="btn"] { cursor: pointer; }

/* Contains word (space-separated) */
[class~="featured"] { border: 2px solid gold; }

Practical Examples

/* Style external links differently */
a[href^="http"]:not([href*="mysite.com"]) {
    color: orange;
}
a[href^="http"]::after {
    content: " ↗";
}

/* Style required inputs */
input[required] {
    border-left: 3px solid red;
}

/* Style different input types */
input[type="email"],
input[type="password"] {
    padding-left: 30px;
    background: url('icon.svg') no-repeat 5px center;
}

Question: How would you select all links that point to PDF files?

Answer: a[href$=".pdf"] — the $= means "ends with".



Responsive Design

Make layouts adapt to different screen sizes.

Viewport Meta Tag

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Required for responsive design. Without it, mobile browsers assume desktop width.

Media Queries

/* Mobile-first approach (recommended) */
.container {
    width: 100%;  /* default: mobile */
}

@media (min-width: 768px) {
    .container {
        width: 750px;  /* tablet and up */
    }
}

@media (min-width: 1024px) {
    .container {
        width: 960px;  /* desktop */
    }
}

Common Breakpoints

  • 576px — small phones → larger phones
  • 768px — phones → tablets
  • 1024px — tablets → laptops
  • 1200px — laptops → desktops

Note: These are guidelines, not rules. Design for your content.

Mobile-First vs Desktop-First

/* Mobile-first: start small, add complexity */
@media (min-width: 768px) { ... }

/* Desktop-first: start big, simplify */
@media (max-width: 768px) { ... }

Mobile-first is recommended — easier to add features than remove them.

Responsive Images

img {
    max-width: 100%;
    height: auto;
}

Images scale down but never larger than their natural size.

Responsive Units

/* Viewport units */
width: 50vw;   /* 50% of viewport width */
height: 100vh; /* 100% of viewport height */

/* Percentage */
width: 80%;    /* 80% of parent */

/* Clamp (min, preferred, max) */
font-size: clamp(1rem, 2.5vw, 2rem);
width: clamp(300px, 50%, 600px);

Hiding/Showing Elements

.mobile-only {
    display: block;
}
.desktop-only {
    display: none;
}

@media (min-width: 768px) {
    .mobile-only {
        display: none;
    }
    .desktop-only {
        display: block;
    }
}

Question: Your nav looks good on desktop but items stack awkwardly on mobile. What's a common solution?

Answer: Use flexbox with flex-direction: column on mobile and flex-direction: row on desktop, or hide the desktop nav and show a hamburger menu on mobile.



CSS Accessibility

Make your styles work for everyone, including users with disabilities.

Focus States

/* Never do this without an alternative */
*:focus { outline: none; }  /* BAD - removes keyboard nav indicator */

/* Do this instead */
button:focus {
    outline: 2px solid #0984e3;
    outline-offset: 2px;
}

/* Or use focus-visible for keyboard-only */
button:focus-visible {
    outline: 2px solid #0984e3;
}

Color Contrast

WCAG Requirements:

  • Normal text: 4.5:1 contrast ratio minimum
  • Large text (18px+ bold or 24px+): 3:1 minimum
  • Use tools like WebAIM Contrast Checker
❌ Low contrast - hard to read
✓ Good contrast - easy to read

Don't Rely on Color Alone

/* Bad: only color indicates error */
.error { color: red; }

/* Good: color + icon/text */
.error {
    color: red;
    border-left: 3px solid red;
}
.error::before {
    content: "⚠ ";
}

Hiding Content Accessibly

/* Visually hidden but available to screen readers */
.sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border: 0;
}

/* Completely hidden (including screen readers) */
.hidden {
    display: none;
}

/* Or */
.hidden {
    visibility: hidden;
}

Reduced Motion

/* Respect user preference for reduced motion */
@media (prefers-reduced-motion: reduce) {
    * {
        animation: none !important;
        transition: none !important;
    }
}

Some users have vestibular disorders where motion causes nausea or dizziness.

Touch Targets

/* Minimum 44x44px for touch targets */
button, a {
    min-height: 44px;
    min-width: 44px;
    padding: 12px;
}


Common Layout Patterns

Navbar

.navbar {
    display: flex;
    justify-content: space-between;
    align-items: center;
    padding: 0 20px;
    height: 60px;
    background: #333;
}

.nav-links {
    display: flex;
    gap: 20px;
}

.nav-links a {
    color: white;
    text-decoration: none;
}

/* Mobile */
@media (max-width: 768px) {
    .nav-links {
        display: none;  /* Hide, show hamburger instead */
    }
}

Card Grid

.card-grid {
    display: flex;
    flex-wrap: wrap;
    gap: 20px;
}

.card {
    flex: 1 1 300px;  /* grow, shrink, min 300px */
    max-width: 400px;
}

Sticky Footer

body {
    min-height: 100vh;
    display: flex;
    flex-direction: column;
}

main {
    flex: 1;  /* takes remaining space */
}

footer {
    /* stays at bottom */
}

Centered Content

/* Horizontal center */
.container {
    max-width: 1200px;
    margin: 0 auto;
    padding: 0 20px;
}

/* Dead center (both axes) */
.center {
    display: flex;
    justify-content: center;
    align-items: center;
    min-height: 100vh;
}
