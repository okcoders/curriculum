Week 1 — HTML Foundations

Review of freeCodeCamp Basic HTML homework. These are the core concepts you should understand.


HTML Boilerplate

Every HTML page needs this structure. The browser uses it to know how to render the page.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>
<body>
    <!-- visible content goes here -->
</body>
</html>
  • <!DOCTYPE html> — tells the browser this is HTML5
  • <html> — root element, wraps everything
  • <head> — metadata, not visible on page (title, charset, links to CSS)
  • <body> — everything the user sees


Headings & Paragraphs

HTML has 6 heading levels. <h1> is the most important, <h6> is the least. Use them in order — don't skip levels.

h1 — Main Heading

h2 — Section Heading

h3 — Subsection

h4 — Sub-subsection

h5 — Minor heading
h6 — Smallest heading

This is a paragraph. The <p> tag creates a block of text with spacing above and below.

Question: Why shouldn't you use <h1> just to make text big?

Answer: Headings have semantic meaning — screen readers and search engines use them to understand page structure. Use CSS for sizing.



Text Formatting

Common inline elements for formatting text:

This is bold — uses <strong> (semantic: important text)

This is italic — uses <em> (semantic: emphasized text)

This is also bold — uses <b> (visual only, no semantic meaning)

This is also italic — uses <i> (visual only, no semantic meaning)

Key difference: <strong> and <em> tell the browser the text is important. Screen readers will emphasize them. <b> and <i> just change how text looks.

Question: When would you use <ol> vs <ul>?

Answer: Use ordered when sequence matters (steps, rankings). Use unordered when order doesn't matter (grocery list, features).

The anchor tag <a> creates hyperlinks. The href attribute holds the URL.

Opens in new tab

Images

Images use the <img> tag. It is a self-closing tag — no closing tag needed.

<img src="photo.jpg" alt="Description of image" width="300">

Required attributes:

  • src — path to the image file
  • alt — describes the image (accessibility + shows if image fails to load)

Question: What happens if you leave out the alt attribute?

Answer: Screen readers can't describe the image to visually impaired users. Also, if the image fails to load, users see nothing instead of a description.



Audio & Video

HTML5 added native media elements so you don't need plugins.

Video

<video src="movie.mp4" controls width="400">
    Your browser does not support video.
</video>

Audio

<audio src="song.mp3" controls>
    Your browser does not support audio.
</audio>

Key attribute: controls — without this, the player is invisible. The text between the tags is a fallback for old browsers.



Iframes

Embeds another webpage inside your page. Common for YouTube videos, maps, etc.

<iframe 
    src="https://www.youtube.com/embed/VIDEO_ID" 
    width="560" 
    height="315" 
    title="Video description"
    allowfullscreen
></iframe>

Always include a title attribute for accessibility.



SVGs

SVG (Scalable Vector Graphics) creates images with code. They scale to any size without losing quality.

<svg width="100" height="100">
    <circle cx="50" cy="50" r="40" fill="blue" />
</svg>

When to use SVG vs <img>: SVGs are best for icons, logos, and simple shapes. Photos should use <img> with jpg/png.



HTML Attributes

Attributes add extra information to elements. They go inside the opening tag.

<tag attribute="value">content</tag>

Common attributes you've seen:

  • id — unique identifier (one per page)
  • class — reusable identifier (many elements can share)
  • src — source file (images, video, audio)
  • href — hyperlink reference (links)
  • alt — alternative text (images)
  • type — input type (forms)
  • target — where to open link (_blank = new tab)
  • width / height — size of element

Question: What's the difference between id and class?

Answer: id must be unique — only one element on the page should have that id. class can be reused on as many elements as you want.



Semantic HTML

Semantic elements describe their meaning. This helps search engines, screen readers, and other developers understand your page structure.

Non-semantic (bad)

<div id="header">...</div>
<div id="nav">...</div>
<div id="main">...</div>
<div id="footer">...</div>

Semantic (good)

<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>

Key semantic elements:

  • <header> — top of page or section (logo, nav)
  • <nav> — navigation links
  • <main> — primary content (only one per page)
  • <section> — thematic grouping of content
  • <article> — self-contained content (blog post, news story)
  • <aside> — sidebar or related content
  • <footer> — bottom of page or section
  • <figure> / <figcaption> — image with caption

Nesting & Indentation

Elements inside other elements are called children. Proper indentation makes code readable.

Bad (no indentation)

<div>
<h1>Title</h1>
<p>Text</p>
<ul>
<li>Item</li>
</ul>
</div>

Good (indented)

<div>
    <h1>Title</h1>
    <p>Text</p>
    <ul>
        <li>Item</li>
    </ul>
</div>

Rule: Every time an element is nested inside another, indent 2 or 4 spaces (pick one and be consistent).



Block vs Inline Elements

Understanding this difference is critical for layout.

Block elements — take full width, start on new line

This is a <div>

This is a <p>

Inline elements — only take needed width, stay on same line

This has a <span> and a <strong> and a <a> inline.

Block: <div>, <p>, <h1>-<h6>, <ul>, <ol>, <li>, <section>, <header>, <footer>

Inline: <span>, <a>, <strong>, <em>, <img>, <code>



HTML Entities

Special characters that HTML reserves need entity codes:

  • &lt; → < (less than)
  • &gt; → > (greater than)
  • &amp; → & (ampersand)
  • &copy; → © (copyright)
  • &nbsp; → non-breaking space

Why? If you type < in your HTML, the browser thinks it's a tag. Entities let you display these characters as text.



Putting It All Together

A well-structured HTML page uses all of these concepts:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Portfolio</title>
</head>
<body>
    <header>
        <h1>Jane Doe</h1>
        <nav>
            <a href="#about">About</a>
            <a href="#projects">Projects</a>
            <a href="#contact">Contact</a>
        </nav>
    </header>

    <main>
        <section id="about">
            <h2>About Me</h2>
            <p>I'm a <strong>web developer</strong> learning HTML.</p>
            <img src="headshot.jpg" alt="Jane smiling">
        </section>

        <section id="projects">
            <h2>Projects</h2>
            <article>
                <h3>Project One</h3>
                <p>Built a recipe page using <em>semantic HTML</em>.</p>
            </article>
        </section>
    </main>

    <footer>
        <p>&copy; 2025 Jane Doe</p>
    </footer>
</body>
</html>
