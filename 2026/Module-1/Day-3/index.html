Week 2 Day 1 — CSS Fundamentals

Review of freeCodeCamp CSS homework. CSS (Cascading Style Sheets) controls how HTML elements look.


How CSS Works

CSS consists of selectors and declarations. The selector targets elements, and declarations set the styles.

selector {
    property: value;
    property: value;
}
h1 {
    color: blue;
    font-size: 24px;
}

This targets all <h1> elements and makes them blue with 24px font size.



Three Ways to Add CSS

1. Inline (in the element)

<p style="color: red;">Red text</p>

Use: Quick fixes, overrides. Avoid: For most styling — hard to maintain.

2. Internal (in the head)

<head>
    <style>
        p { color: red; }
    </style>
</head>

Use: Single-page styles, demos, emails. Avoid: Multi-page sites.

3. External (separate file)

<head>
    <link rel="stylesheet" href="styles.css">
</head>

Use: Most projects. Why: Reusable across pages, easier to maintain.



CSS Selectors

Selectors determine which elements get styled.

Element Selector

p {
    color: gray;
}

Targets all <p> elements on the page.

Class Selector (uses .)

.highlight {
    background-color: yellow;
}

<p class="highlight">I'm highlighted</p>
<p>I'm not</p>

Reusable: Multiple elements can share the same class.

ID Selector (uses #)

#main-title {
    font-size: 32px;
}

<h1 id="main-title">Welcome</h1>

Unique: Only ONE element per page should have each ID.

Descendant Selector

.card p {
    color: gray;
}

Targets <p> elements inside elements with class card.

Multiple Selectors

h1, h2, h3 {
    font-family: Georgia, serif;
}

Applies the same style to multiple elements. Separate with commas.

Question: What's the difference between .card p and .card, p?

Answer: .card p = paragraphs inside .card only. .card, p = all .card elements AND all paragraphs.



CSS Specificity

When multiple rules target the same element, specificity determines which wins.

Specificity hierarchy (highest to lowest):

  1. Inline styles — style="..." (1000 points)
  2. ID selectors — #id (100 points)
  3. Class selectors — .class (10 points)
  4. Element selectors — p, div (1 point)
<style>
    p { color: black; }           /* 1 point */
    .text { color: blue; }        /* 10 points */
    #intro { color: red; }        /* 100 points */
</style>

<p id="intro" class="text">What color am I?</p>

Answer: Red — ID selector wins.

Question: If you can't figure out why your style isn't applying, what should you check?

Answer: Check if a more specific selector is overriding it. Use browser DevTools to see which rules are applied/overridden.



Colors

CSS supports multiple ways to define colors.

Named Colors

red
blue
green
tomato
steelblue

CSS has 140+ named colors. Good for quick prototyping.

Hex Colors

color: #ff0000;  /* red */
color: #00ff00;  /* green */
color: #0000ff;  /* blue */
color: #000000;  /* black */
color: #ffffff;  /* white */
color: #333;     /* shorthand for #333333 (dark gray) */

6 digits: #RRGGBB — each pair is 00-FF (0-255 in hex).

RGB / RGBA

color: rgb(255, 0, 0);       /* red */
color: rgb(0, 128, 0);       /* green */
background: rgba(0, 0, 0, 0.5);  /* black at 50% opacity */

rgba adds alpha channel (0 = transparent, 1 = opaque).

HSL / HSLA

color: hsl(0, 100%, 50%);    /* red */
color: hsl(120, 100%, 50%);  /* green */
color: hsl(240, 100%, 50%);  /* blue */

Hue (0-360°), Saturation (0-100%), Lightness (0-100%). Easier to adjust than hex.

Question: You want a button that's red but slightly transparent. Which color format would you use?

Answer: rgba(255, 0, 0, 0.8) or hsla(0, 100%, 50%, 0.8) — any format with alpha channel.



Units

CSS has absolute and relative units.

Absolute Units

  • px — pixels (most common for web)
  • pt — points (print)
  • cm, mm, in — physical units (rarely used on web)

Relative Units

  • em — relative to parent's font-size
  • rem — relative to root (<html>) font-size
  • % — percentage of parent
  • vw — 1% of viewport width
  • vh — 1% of viewport height
html { font-size: 16px; }

.container {
    font-size: 20px;
}

.container p {
    font-size: 1em;   /* 20px (parent's size) */
    font-size: 1rem;  /* 16px (root's size) */
}

Tip: Use rem for consistent sizing, em when you want scaling relative to parent.



The Box Model

Every element is a box with four layers: content, padding, border, margin.

Content

← padding →

.box {
    width: 200px;
    padding: 20px;      /* inside the border */
    border: 5px solid black;
    margin: 10px;       /* outside the border */
}

Calculating Total Width (default box-sizing)

Total width = width + padding-left + padding-right + border-left + border-right

.box {
    width: 200px;
    padding: 20px;
    border: 5px solid black;
}
/* Total width = 200 + 20 + 20 + 5 + 5 = 250px */

box-sizing: border-box

.box {
    box-sizing: border-box;
    width: 200px;
    padding: 20px;
    border: 5px solid black;
}
/* Total width = 200px (padding & border included) */
/* Content width = 200 - 20 - 20 - 5 - 5 = 150px */

Best practice: Use border-box — width means total width, much easier to work with.

/* Apply border-box to everything (common reset) */
*, *::before, *::after {
    box-sizing: border-box;
}

Margin Collapse

.box1 { margin-bottom: 20px; }
.box2 { margin-top: 30px; }

/* Space between them is 30px, NOT 50px */

Vertical margins collapse — the larger one wins. Horizontal margins do NOT collapse.



Pseudo-Classes

Style elements based on their state.

a:link { color: blue; }      /* unvisited */
a:visited { color: purple; } /* visited */
a:hover { color: red; }      /* mouse over */
a:active { color: orange; }  /* being clicked */

button:hover {
    background-color: darkblue;
    cursor: pointer;
}

input:focus {
    border-color: blue;
    outline: none;
}

Structural Pseudo-Classes

li:first-child { font-weight: bold; }
li:last-child { color: gray; }
li:nth-child(odd) { background: #f5f5f5; }
li:nth-child(3) { color: red; }  /* third item */


Pseudo-Elements

Style parts of an element. Use :: (double colon).

p::first-line {
    font-weight: bold;
}

p::first-letter {
    font-size: 2em;
    float: left;
}

/* Add content before/after an element */
.required::after {
    content: " *";
    color: red;
}

blockquote::before {
    content: open-quote;
}
blockquote::after {
    content: close-quote;
}

Question: What's the difference between :hover and ::before?

Answer: :hover is a pseudo-class (state). ::before is a pseudo-element (part of element). Single colon for classes, double for elements.



Styling Forms

Forms need special attention for usability.

/* Basic input styling */
input, select, textarea {
    width: 100%;
    padding: 10px;
    margin-bottom: 15px;
    border: 1px solid #ccc;
    border-radius: 4px;
    font-size: 16px;
}

/* Focus state */
input:focus {
    border-color: #0984e3;
    outline: none;
    box-shadow: 0 0 5px rgba(9, 132, 227, 0.3);
}

/* Placeholder text */
input::placeholder {
    color: #999;
    font-style: italic;
}

/* Invalid state */
input:invalid {
    border-color: red;
}

/* Disabled state */
input:disabled {
    background-color: #eee;
    cursor: not-allowed;
}

Styling Buttons

button {
    background-color: #0984e3;
    color: white;
    padding: 12px 24px;
    border: none;
    border-radius: 4px;
    cursor: pointer;
    font-size: 16px;
}

button:hover {
    background-color: #0773c7;
}

button:active {
    transform: translateY(1px);
}

Styling Checkboxes & Radio Buttons

/* Custom checkbox using pseudo-elements */
input[type="checkbox"] {
    width: 20px;
    height: 20px;
    accent-color: #0984e3;  /* modern browsers */
}

accent-color is the modern way to color checkboxes/radios. Older techniques hide the input and style a ::before pseudo-element.



Common CSS Properties Reference

Text & Font

font-family: Arial, sans-serif;
font-size: 16px;
font-weight: bold;        /* or 100-900 */
font-style: italic;
text-align: left | center | right | justify;
text-decoration: underline | none | line-through;
text-transform: uppercase | lowercase | capitalize;
line-height: 1.5;
letter-spacing: 2px;

Background

background-color: #f5f5f5;
background-image: url('image.jpg');
background-size: cover | contain | 100px 200px;
background-position: center | top left;
background-repeat: no-repeat | repeat-x | repeat-y;

/* Shorthand */
background: #f5f5f5 url('image.jpg') no-repeat center/cover;

Border

border: 1px solid #333;
border-width: 1px;
border-style: solid | dashed | dotted | none;
border-color: #333;
border-radius: 4px;       /* rounded corners */
border-radius: 50%;       /* circle */

Display

display: block;        /* full width, new line */
display: inline;       /* flows with text, no width/height */
display: inline-block; /* flows with text, but accepts width/height */
display: none;         /* removes from page */
visibility: hidden;    /* invisible but takes up space */


CSS Reset / Normalize

Browsers have different default styles. A reset creates consistency.

/* Minimal reset */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

/* Or use a full reset like normalize.css */
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/normalize/8.0.1/normalize.min.css">


Debugging CSS

Browser DevTools are essential for debugging CSS.

How to use DevTools:

  1. Right-click an element → Inspect
  2. See applied styles in the Styles panel
  3. Crossed-out styles are being overridden
  4. Click the box model diagram to see spacing
  5. Toggle styles on/off with checkboxes
  6. Edit values live to test changes

Common issues to check:

  • Is your selector correct? (typo in class name?)
  • Is a more specific rule overriding yours?
  • Is the element display type correct? (block vs inline)
  • Is box-sizing causing unexpected sizes?
  • Are margins collapsing?
